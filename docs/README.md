# Docs - 6周工作记录（按周）

本目录根据根目录 `README.md` 中的「6周详细日程」整理而来，目标是把每周要做什么、怎么做、为什么这么做、容易踩哪些坑、怎么排查与解决写清楚，并配套流程图，方便复盘与演示。

## 使用方式

- 每周开始前：先读对应周文档的「目标/交付物」与「风险点」。
- 每天推进时：按「操作步骤」逐项完成并记录实际差异。
- 遇到问题：优先看「常见问题与解决」，再回到对应组件的日志/配置。

## 统一约定（建议）

- **文档记录粒度**：以「可重复执行」为准（命令 + 配置位置 + 预期结果）。
- **验收标准**：每周至少包含 1 条可演示的端到端路径。
- **排障方法论**：先确认网络/端口/健康检查，再确认配置/凭证，最后看业务逻辑。
